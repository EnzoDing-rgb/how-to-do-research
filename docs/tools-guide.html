1 Vibero · 氛围阅读

先读文献、扩展上下文 — 非线性阅读，先见森林再见树木

是什么：AI 论文阅读器，把静态 PDF 变成可交互的画布；支持结构解析、点选追问、笔记留在画布上。

  • 官网（中文）： https://vibero.dev/cn
  • 国际版： https://vibero.dev

安装 / 获取：

  1. 打开 vibero.dev/cn
  2. 点击页面上的 「下载 Vibero」，按你的系统（macOS / Windows）下载安装包
  3. 安装后注册账号，即可导入 PDF 开始阅读

推荐用法：

  • 打开一篇论文 → 等待 VIBE 解析 生成左侧结构树
  • 先浏览整体论点，再点选段落向 AI 追问
  • 把有价值的问答拖回画布，形成自己的阅读笔记层
国内使用大部分功能不需要科学上网；仅在使用 GitHub 代码库问答时，境内可能需要 VPN。

2 OpenCode · AI 编程 Agent

再落地执行 — 用 Agent 跑脚本、处理数据、自动化研究工序

是什么：开源 AI Agent，可在终端、桌面或 IDE 里读项目、改文件、多步执行任务；内置免费模型，也可接入 GPT、Gemini 等。

  • 下载页（中文）： https://opencode.ai/zh/download
  • 官网： https://opencode.ai
  • 文档： https://opencode.ai/docs
  • GitHub： github.com/anomalyco/opencode

安装方式（任选其一）：

终端版 — 全平台通用（推荐课堂统一用这个）：

curl -fsSL https://opencode.ai/install | bash

其他常见方式：

# npm
npm i -g opencode-ai

# Homebrew（macOS / Linux）
brew install anomalyco/tap/opencode

# 桌面版 Beta（macOS / Windows / Linux）
# 见下载页直接下载 .dmg / .exe / .deb
brew install --cask opencode-desktop   # macOS

完整下载选项（桌面版、VS Code / Cursor 插件等）见官方下载页：
opencode.ai/zh/download

安装后第一步：

  1. 终端输入 opencode 启动（或打开桌面版）
  2. 按提示连接模型（可用内置免费模型，或填入自己的 API Key）
  3. 在一个项目目录里，用自然语言描述任务，例如：「帮我对这份 CSV 做描述统计」
Windows 用户：官方推荐用 WSL 获得最佳体验；也可直接下载桌面版 Beta。

3 Research Assistant · 科研全流程 Skill

最后串起来 — 把读和做编排成一条可重复的研究管线

是什么：一套科研全流程 Skill，覆盖研究设计 → 文献搜索 → 结构写作 → 同行评审 → 修订交付；带质量门禁和弱点路由，不过关不往下走。

  • GitHub 仓库： github.com/EnzoDing-rgb/research-assistant
  • 核心 Skill 文件： research-skills.md
  • 示例 · 世界模型综述： examples/world-model

获取代码：

git clone https://github.com/EnzoDing-rgb/research-assistant.git
cd research-assistant

在 OpenCode 里使用（推荐）：

  1. 克隆仓库后，把 research-skills.md 复制到你项目的 Skill 目录（OpenCode 会读取项目级或全局 Skill）
  2. 在 OpenCode 里打开你的研究项目目录
  3. 对 Agent 说：「按照 research-skills 的流程，帮我做一篇关于 XXX 的文献综述」
  4. Agent 会按阶段推进：设计 → 文献 → 结构 → 评审 → 交付，每阶段有明确产出

在 Claude Code 里使用（可选）：

  1. 需先安装 Claude Code CLI
  2. 进入克隆好的 research-assistant 目录
  3. 运行一键安装依赖：
    bash setup.sh
    会安装 Academic Research Skills、Ralph Loop 等插件（OpenSpec 可选）
  4. 在 Claude Code 中输入：「用社科研究助手」+ 你的研究问题

管线架构（了解即可）：

研究设计 → 文献搜索 → 实验设计 → 图表 & 结构 → 同行评审 → 路由修订 → 交付
   ↑           ↑           ↑            ↑           │
   └───────────┴───────────┴────────────┴───────────┘
                    弱点路由表（定向修复，非盲目重试）
和 OpenCode 的关系：Research Assistant 本质是 Skill（方法论 + 工作流指令），OpenCode 是执行它的 Agent 环境。把 Skill 装进 OpenCode，就和 Vibero 的阅读能力拼成完整闭环。

推荐使用顺序

① Vibero 读文献、建上下文
↓
② OpenCode 跑数据、写脚本、做整理
↓
③ Research Assistant 编排全流程、产出综述 / 报告
↓
（进阶彩蛋：Auto Research 全自动科研）
